Step Three: Evaluations

Now, evaluate the prototypes to see if they are effective and where changes need to be made.

First, we had participants interact with our prototype for a few minutes. They would make an action on the screen, and the researcher would describe to them what visible results that action was meant to have, in many cases this would mean directing them to a new screen. To simplify the testing process, we came up with 11 questions, ten of which targeted one of the Nielsen heuristics, and one which was mean to encourage discussion with the participant. Participants answered these questions about the experience they had with the prototype.

After interviewing multiple participants, we started noticing some redundancy in their answers. Below are the most common themes among the answers we were given.

  1. Can you tell where you are in the app at all times?

    Yes, but not always on the date picker screens, especially if they've looked away for a while.

  2. Do you think you would need to learn any new terminology or iconography to use the app?

    No, it uses the same icons that I'm used to from other apps. Navigation with swiping is sometimes unusual, though, especially on the typing screen.

  3. Do you feel you can easily undo a mistake you’ve made, like typing something you didn’t want to, or giving the wrong information for an event?

    No, sometimes I'll get stuck in a menu that I won't be able to leave. (Date picker)

  4. Do you think the system can help you avoid making errors?

    No, there are no confirmation dialogs to help me avoid making errors.

  5. Does the system make it clear that an error has occurred?

    No, there are no error dialogs.

  6. Is the same ‘language’ spoken across the entire app?

    Yes, although the time is displayed differently on a couple screens.

  7. Do you think you can quickly recognize all available actions, rather than needing to memorize them?

    No, navigation around pages can be challenging unless you've memorized all the actions.

  8. Does the system continually reward its use such that experienced users are able to gain more from the app?

    No, except that experienced users will understand how to navigate, but that should have been intuitive.

  9. Is there any unnecessary information?

    No, in fact, there is not enough information, an animation describing how to navigate could be helpful.

  10. If you encountered a bug, would it be easy to find help to resolve?

    No, I would have to Google it or go to the settings menu, but it is challenging to get to the settings menu sometimes.

  11. Is there anything else you'd like to say about the app? Anything you'd like to change?

    The long and short of it is: it's hard to navigate through the app and some things are inconsistent.

Planned Changes

All pages will use pinch to exit, instead of swiping down. The animations will reflect this. On any page, they should be able to long press to bring up button versions of the available actions.

The 'Feb 2019' box should open the 'jump to date' feature. when tapped

Today View

Event Details View

  • Add a save button at the bottom.
  • Add a bell icon on the notification timer.
  • Have seperate inputs for start & end time, or start & duration.
  • Pinch to exit, should show confirmation dialog for saving or not.
  • Add a way to categorize the events by color.

  • Check mark to save
